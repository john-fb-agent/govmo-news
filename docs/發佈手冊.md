# 發佈手冊

## 📋 發佈流程

本文件說明如何發佈 govmo-news 的新版本。

## 發佈前準備

### 1. 確認發佈內容

- [ ] 所有功能已完成並測試
- [ ] 所有 bug 已修復
- [ ] 文檔已更新
- [ ] 變更日誌已撰寫

### 2. 運行測試

```bash
# 運行所有測試
cd /home/js/.openclaw/workspace/github-repos/govmo-news
pytest tests/ -v

# 檢查代碼質量（如已配置）
flake8 src/
```

### 3. 更新版本號

選擇版本號類型：

- **MAJOR.MINOR.PATCH** (例如：1.0.0)
  - **MAJOR**: 不兼容的 API 變更
  - **MINOR**: 向後兼容的功能新增
  - **PATCH**: 向後兼容的問題修復

更新以下文件中的版本號：

**README.md:**
```markdown
**版本：** 1.0.0
```

**docs/專案說明.md:**
```markdown
**版本：** 1.0.0
```

### 4. 更新變更日誌

在 `docs/變更日誌.md`（待建立）中添加：

```markdown
## [1.0.0] - 2026-04-14

### Added
- 初始版本
- RSS 新聞抓取功能
- XML 轉 JSON 功能
- 自動去重功能
- 定時任務支援

### Changed
- 無

### Fixed
- 無

### Known Issues
- 待確認事項見 docs/專案說明.md
```

## 發佈步驟

### 步驟 1：提交變更

```bash
cd /home/js/.openclaw/workspace/github-repos/govmo-news

# 添加所有變更
git add .

# 提交
git commit -m "release: 發佈版本 1.0.0"
```

### 步驟 2：推送到 GitHub

```bash
git push origin main
```

### 步驟 3：建立 Git Tag

```bash
# 建立標籤
git tag -a v1.0.0 -m "發佈版本 1.0.0"

# 推送標籤
git push origin v1.0.0
```

### 步驟 4：建立 GitHub Release

1. 訪問：https://github.com/john-fb-agent/govmo-news/releases
2. 點擊 "Draft a new release"
3. 選擇標籤：`v1.0.0`
4. 填寫發佈說明：

```markdown
# govmo-news v1.0.0

## 新功能

- ✅ RSS 新聞抓取
- ✅ XML 轉 JSON
- ✅ 自動去重
- ✅ 定時任務

## 安裝

```bash
git clone https://github.com/john-fb-agent/govmo-news.git
cd govmo-news
pip install -r requirements.txt
```

## 使用

```bash
python src/backend/fetch_news.py
```

## 變更詳情

參見 [變更日誌](docs/變更日誌.md)
```

5. 點擊 "Publish release"

## 發佈後驗證

### 1. 驗證 Release

- [ ] GitHub Release 頁面顯示正確
- [ ] Tag 已建立
- [ ] 變更日誌正確

### 2. 測試安裝

```bash
# 在新目錄測試安裝
cd /tmp
git clone https://github.com/john-fb-agent/govmo-news.git
cd govmo-news
git checkout v1.0.0
pip install -r requirements.txt
python src/backend/fetch_news.py
```

## 緊急回滾

如發佈後發現嚴重問題：

### 步驟 1：刪除 Tag 和 Release

1. 刪除 GitHub Release
2. 刪除 Tag：

```bash
git tag -d v1.0.0
git push origin --delete v1.0.0
```

### 步驟 2：回滾代碼

```bash
# 回滾到上一個穩定版本
git revert HEAD
git push origin main
```

## 定期發佈建議

### 發佈頻率

- **PATCH 版本：** 按需（bug 修復）
- **MINOR 版本：** 每月或每兩個月（新功能）
- **MAJOR 版本：** 每年或不兼容變更時
