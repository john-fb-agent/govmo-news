# 已知問題

本文檔記錄 govmo-news 的已知問題和限制。

## 🔴 嚴重問題

目前無嚴重問題。

## 🟡 已知限制

### 1. RSS Feed 依賴外部服務

**問題：** 系統完全依賴澳門政府新聞局的 RSS API

**影響：**
- 如 API 不可用，系統無法抓取新聞
- 如 API 格式變更，解析可能失敗

**緩解措施：**
- 添加錯誤處理和重試機制（待實施）
- 監控 API 可用性（待實施）

**追蹤 Issue：** 待建立

### 2. 無 Web 介面

**問題：** 目前僅有命令列介面

**影響：**
- 非技術用戶難以使用
- 無法在線查看新聞

**計劃：** 未來版本可能添加 Web 介面

**追蹤 Issue：** 待建立

### 3. 去重邏輯簡單

**問題：** 目前僅基於 URL + 標題 + 時間的 MD5 哈希

**影響：**
- 如新聞更新但 URL 不變，可能被誤判為重複
- 如不同新聞有相同標題，可能無法去重

**計劃：** 評估更複雜的去重策略

**追蹤 Issue：** 待建立

## 🟢 輕微問題

### 1. 日誌文件可能過大

**問題：** 日誌文件無輪轉機制

**影響：**
- 長期運行後日誌文件可能佔用大量磁碟空間

**緩解措施：**
- 手動清理：`rm data/*.log`
- 設置 logrotate（見 deployment/README.md）

**計劃：** 添加自動日誌輪轉

**追蹤 Issue：** 待建立

### 2. 無監控告警

**問題：** 抓取失敗時無自動通知

**影響：**
- 可能長時間未發現抓取失敗

**緩解措施：**
- 定期檢查日誌
- 檢查 `data/processed/` 是否有新數據

**計劃：** 添加監控和告警機制

**追蹤 Issue：** 待建立
