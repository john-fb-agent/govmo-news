# 開發規則

## 📋 編碼規範

### Python 編碼風格

遵循 [PEP 8](https://pep8.org/) Python 編碼規範：

- **縮排：** 4 個空格（不使用 Tab）
- **行長：** 最多 100 字符
- **編碼：** UTF-8
- **_import_ 順序：** 標準庫 → 第三方庫 → 本地模組

### 命名規範

```python
# 變數和函數：snake_case
news_items = []
def fetch_news():
    pass

# 類別：PascalCase
class RSSParser:
    pass

# 常數：UPPER_CASE
MAX_RETRIES = 3
RSS_URL = "https://..."

# 私有方法：底線前綴
def _internal_helper():
    pass
```

### 文件結構

每個 Python 文件應包含：

```python
#!/usr/bin/env python3
# -*- coding: utf-8 -*-
"""
模組說明（簡短描述模組用途）
"""

import os
import sys
from typing import List, Dict

# 程式碼...
```

## 📝 文件規範

### 文件命名

- **Python 文件：** `snake_case.py`
- **文檔文件：** `繁體中文.md`
- **腳本文件：** `動作 - 目標.sh`（如 `setup-cron.sh`）

### 文件位置

```
govmo-news/
├── src/           # 所有原始碼

├── deployment/    # 部署相關
├── tests/         # 測試相關
├── docs/          # 文檔
└── .gitignore     # Git 忽略規則
```

## 🔄 Git 工作流

### 分支策略

```
main          # 主分支，隨時可部署
├── develop   # 開發分支（可選）
└── feature/* # 功能分支
```

### Commit 訊息格式

```
<類型>: <簡短描述>

[可選的詳細說明]

[可選的相關 Issue]
```

**類型：**
- `feat`: 新功能
- `fix`: 修復 bug
- `docs`: 文檔更新
- `style`: 代碼格式（不影響功能）
- `refactor`: 重構（不改變行為）
- `test`: 添加測試
- `chore`: 構建過程或輔助工具變動

**範例：**
```
feat: 添加新聞去重功能

- 實現基於 MD5 哈希的去重邏輯
- 添加 .processed_ids.json 數據庫
- 更新 fetch_news.py 整合去重檢查

Closes #12
```

### Pull Request 流程

1. 從 `main` 建立功能分支
2. 開發並提交
3. 推送分支到 GitHub
4. 建立 Pull Request
5. 代碼審查
6. 合併到 `main`

## 🧪 測試規範

### 測試目錄結構

```
tests/
├── README.md         # 測試說明
├── test_rss_parser.py  # 解析器測試（待添加）
└── test_dedup.py     # 去重測試（待添加）
```

### 測試命名

```python
def test_fetch_success():
    """測試抓取成功的情況"""
    pass

def test_fetch_network_error():
    """測試網絡錯誤的情況"""
    pass

def test_dedup_duplicate_news():
    """測試重複新聞被正確過濾"""
    pass
```

### 運行測試

```bash
# 所有測試
pytest tests/

# 特定測試
pytest tests/test_rss_parser.py

# 帶覆蓋率
pytest --cov=src tests/
```

## 📦 依賴管理

### 添加新依賴

1. 測試依賴是否必要
2. 添加到 `requirements.txt`
3. 更新文檔中的依賴說明
4. 提交變更

### requirements.txt 格式

```txt
# 套件名稱>=最低版本
feedparser>=6.0.0
requests>=2.28.0

# 可選依賴（測試用）
pytest>=7.0.0
```

## 🔒 安全規範

### 敏感數據處理

**禁止：**
- ❌ 在代碼中硬編碼 API 密钥
- ❌ 提交密碼或令牌到 Git
- ❌ 在日誌中記錄敏感信息

**建議：**
- ✅ 使用環境變數
- ✅ 使用配置文件（加入 .gitignore）
- ✅ 使用密鑰管理服務

### 環境變數範例

```bash
# .env 文件（不提交到 Git）

LOG_LEVEL=INFO
```

## 🐛 錯誤處理規範

### 日誌級別使用

```python
import logging

logging.debug("詳細的調試信息")     # 開發時使用
logging.info("一般信息")           # 正常運行信息
logging.warning("警告，但不影響運行")  # 需要注意但不需立即處理
logging.error("錯誤，影響部分功能")   # 需要調查
logging.critical("嚴重錯誤，系統無法運行")  # 需要立即處理
```

### 異常處理

```python
# 好的做法
try:
    result = fetch_rss()
except requests.exceptions.Timeout:
    logger.error("RSS 請求超時")
    return None
except Exception as e:
    logger.exception(f"未知錯誤：{e}")
    return None

# 避免
try:
    result = fetch_rss()
except:  # ❌ 不要捕獲所有異常
    pass  # ❌ 不要靜默失敗
```
